# Phase 32-01: Provider Selection Core Implementation

## Goal
Refactor Agent42's core provider selection to prioritize Claude Code Subscription with Synthetic.new as fallback, and remove StrongWall references.

## Requirements Addressed
- PROVIDER-01: Claude Code Subscription is the primary provider for all agent executions
- PROVIDER-02: Synthetic.new API is the fallback provider when Claude Code Subscription is unavailable or task violates CC Subscription TOS
- PROVIDER-03: Remove all StrongWall.ai references and integration from the codebase
- PROVIDER-04: Other LLM providers (Anthropic, OpenRouter, etc.) are used only when CC Subscription and Synthetic.new are unavailable
- PROVIDER-05: Provider selection follows hierarchy: Claude Code Subscription → Synthetic.new → Other API keys

## Implementation Plan

### 1. Core Provider Selection Logic Refactor

#### 1.1. Update TieredRoutingBridge.resolve()
Modify the provider selection logic in `core/tiered_routing_bridge.py`:
- Check for Claude Code Subscription availability first
- Fall back to Synthetic.new when CC Subscription is unavailable
- Remove StrongWall references
- Maintain fallback to other providers when both CC Subscription and Synthetic.new are unavailable

#### 11.2. Update AgentRuntime._build_env()
Modify the environment building logic in `core/agent_runtime.py`:
- Add support for Claude Code Subscription environment variables
- Update provider URL mappings
- Remove StrongWall-specific configurations

### 2. Configuration Updates

#### 2.1. Update Settings
Modify `core/config.py` to add:
- New configuration fields for Claude Code Subscription
- Remove StrongWall-specific configuration fields
- Update documentation and .env.example

#### 2.2. Update .env.example
Update the example environment file to:
- Add Claude Code Subscription configuration
- Remove StrongWall configuration
- Update documentation

### 3. Provider Model Mapping Updates

#### 3.1. Update PROVIDER_MODELS
Modify `core/agent_manager.py` to:
- Add Claude Code Subscription to PROVIDER_MODELS
- Remove StrongWall from PROVIDER_MODELS
- Update model mappings for Synthetic.new if needed

### 4. Remove StrongWall References

#### 4.1. Code Cleanup
Search and remove all references to StrongWall throughout the codebase:
- Configuration files
- Provider selection logic
- Documentation
- Tests

#### 4.2. Environment Variable Cleanup
Remove StrongWall-related environment variables from:
- .env.example
- Configuration loading
- Documentation
