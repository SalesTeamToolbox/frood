**src/session.ts** (per Research Pattern 4, CORRECTING D-08):

Import `AdapterSessionCodec` type from `@paperclipai/adapter-utils`.

CRITICAL CORRECTION from Research: The `AdapterSessionCodec` interface uses `serialize`/`deserialize` methods, NOT `encode`/`decode` as D-08 mentions. The session state is `Record<string, unknown>`, not a base64 string. Paperclip stores the serialized Record directly.

```typescript
import type { AdapterSessionCodec } from "@paperclipai/adapter-utils";

export const sessionCodec: AdapterSessionCodec = {
  serialize(params: Record<string, unknown> | null): Record<string, unknown> | null {
    if (!params) return null;
    return { ...params };  // shallow copy — preserves all fields including unknown ones (D-10)
  },

  deserialize(raw: unknown): Record<string, unknown> | null {
    if (raw == null || typeof raw !== "object" || Array.isArray(raw)) return null;
    return { ...(raw as Record<string, unknown>) };  // shallow copy (D-10)
  },

  getDisplayId(params: Record<string, unknown> | null): string | null {
    if (!params?.agentId || typeof params.agentId !== "string") return null;
    return `agent42:${params.agentId}`;
  },
};
```

The serialize/deserialize pair is intentionally simple (near-identity): Paperclip handles serialization to/from its own storage. The codec's role is defensive parsing (deserialize ensures the unknown input is a valid Record) and forward-compatibility (spread preserves unknown fields per D-10).

The `sessionKey` string in `AdapterConfig` (Pydantic model) is a separate concern — the adapter builds that string using `JSON.stringify(sessionState)` in execute() when constructing the sidecar POST body. The codec itself works with Records.

`execute(ctx: AdapterExecutionContext): Promise<AdapterExecutionResult>`:

1. Parse adapter config: `const config = parseSidecarConfig(ctx.agent.adapterConfig)` (Pitfall 2 — defensive parsing of `unknown`)

2. Create client: `const client = new Agent42Client(config.sidecarUrl, config.bearerToken)`

3. Resolve agentId (ADAPT-04, D-13, D-14):
   - `const agentId = config.agentId || ctx.agent.id`
   - This ensures the Agent42 agent UUID is used when configured, falling back to Paperclip's agent ID

4. Extract and validate wakeReason (ADAPT-03, D-11, D-12):
   - `const wakeReason = (ctx.context?.wakeReason as string) ?? "heartbeat"`
   - If not in KNOWN_WAKE_REASONS set, log warning via `ctx.onLog("stderr", ...)`  but do NOT throw

5. Decode existing session state (ADAPT-05):
   - `const sessionState = ctx.runtime.sessionParams ? (sessionCodec.deserialize(ctx.runtime.sessionParams) ?? {}) : {}`

6. Build sidecar POST body (maps Paperclip context to Agent42 sidecar format):
   ```typescript
   const sidecarBody: SidecarExecuteRequest = {
     runId: ctx.runId,
     agentId,                          // top-level (D-14)
     companyId: ctx.agent.companyId,
     taskId: (ctx.context?.taskId as string) ?? "",
     wakeReason,                       // from ctx.context (D-11)
     context: ctx.context ?? {},       // full Paperclip context dict passthrough
     adapterConfig: {
       sessionKey: JSON.stringify(sessionState),  // string encoding for Pydantic sessionKey field
       memoryScope: config.memoryScope ?? "agent",
       preferredProvider: config.preferredProvider ?? "",
       agentId,                        // also in adapterConfig (D-14)
     },
   };
   ```

7. Call sidecar:
   ```typescript
   try {
     const resp = await client.execute(sidecarBody);
     // Build updated session state
     const newSessionState: Record<string, unknown> = {
       ...sessionState,
       agentId,
       lastRunId: ctx.runId,
       executionCount: ((sessionState.executionCount as number) ?? 0) + 1,
     };
     return {
       exitCode: 0,
       signal: null,
       timedOut: false,
       summary: `Accepted (runId=${ctx.runId}, deduplicated=${resp.deduplicated})`,
       sessionParams: sessionCodec.serialize(newSessionState),
       sessionDisplayId: `run:${ctx.runId}`,
     };
   } catch (err) {
     const message = err instanceof Error ? err.message : String(err);
     return {
       exitCode: 1,
       signal: null,
       timedOut: false,
       errorMessage: message,
       summary: `Failed: ${message}`,
     };
   }
   ```

`testEnvironment(ctx: AdapterEnvironmentTestContext): Promise<AdapterEnvironmentTestResult>` (per Research Pattern 3):

1. Parse config: `const config = parseSidecarConfig(ctx.config)`
2. Initialize checks array and status
3. If `!config.sidecarUrl`: push error check `{ code: "missing_sidecar_url", level: "error", message: "sidecarUrl is not configured", hint: "Set sidecarUrl in adapter config" }`, set status "fail"
4. Else: create client, call `client.health()`:
   - On success with status "ok": push info check `{ code: "sidecar_reachable", level: "info", message: "Agent42 sidecar reachable at {url}" }`
   - Additionally check `health.memory.available`, `health.qdrant.available` — add warn checks if false
   - On error: push error check `{ code: "sidecar_unreachable", level: "error", message: "Cannot reach Agent42 sidecar: {err}", hint: "Verify Agent42 is running at {url}" }`, set status "fail"
5. Return `{ adapterType: ctx.adapterType, status, checks, testedAt: new Date().toISOString() }`
