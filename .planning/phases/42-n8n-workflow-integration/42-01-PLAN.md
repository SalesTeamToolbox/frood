    **execute() method** — action router (per D-04 through D-07):
    ```python
    async def execute(self, action: str = "", **kwargs) -> ToolResult:
        settings = self._get_settings()
        if not settings.n8n_url or not settings.n8n_api_key:
            return ToolResult(success=False, error="N8N not configured. Set N8N_URL and N8N_API_KEY environment variables.")
        match action:
            case "list":    return await self._list_workflows(settings)
            case "trigger": return await self._trigger_workflow(settings, **kwargs)
            case "status":  return await self._get_status(settings, **kwargs)
            case "output":  return await self._get_output(settings, **kwargs)
            case _:         return ToolResult(success=False, error=f"Unknown action '{action}'. Use: list, trigger, status, output")
    ```

    **_list_workflows()** (per D-04):
    - `GET {base_url}/workflows` with `X-N8N-API-KEY` header, timeout=10s (per D-17)
    - Paginate using cursor pattern from RESEARCH.md Section 2.1
    - Return JSON with `[{id, name, active, tags}]` for each workflow
    - Handle httpx errors gracefully, return ToolResult(success=False, error=str(e))

    **_trigger_workflow()** (per D-05, REVISED from D-16 per RESEARCH.md):

    CRITICAL: There is NO `POST /api/v1/workflows/{id}/execute` endpoint. The trigger MUST use webhook URLs.

    1. Validate N8N URL through UrlPolicy SSRF check (per D-21): `from core.url_policy import UrlPolicy; policy = UrlPolicy(); allowed, reason = policy.check(settings.n8n_url); if not allowed: return error`
    2. Check rate limit (per D-22): Use tool_rate_limiting pattern -- track calls in a module-level dict, enforce 10 trigger calls per 60 seconds
    3. `GET {base_url}/workflows/{workflow_id}` with auth headers, timeout=10s -- fetch workflow JSON to find webhook node
    4. Find webhook node: `next((n for n in workflow["nodes"] if n["type"] == "n8n-nodes-base.webhook"), None)`
    5. If no webhook node found, return `ToolResult(success=False, error="Workflow has no webhook trigger node. Add a Webhook node to the workflow in N8N.")`
    6. Extract path: `webhook_node["parameters"]["path"]`
    7. Build webhook URL: `f"{settings.n8n_url}/webhook/{path}"` (production URL for active workflows)
    8. `POST {webhook_url}` with `json=input_data or {}`, timeout=30s (per D-17), NO auth header (webhooks don't need API key)
    9. Return `ToolResult(output=resp.text, success=resp.is_success)`

    **_get_status()** (per D-06):
    - `GET {base_url}/executions/{execution_id}?includeData=true` with auth headers, timeout=10s
    - Extract `status`, `finished`, `startedAt`, `stoppedAt` from response
    - Return JSON summary string

    **_get_output()** (per D-07):
    - `GET {base_url}/executions/{execution_id}?includeData=true` with auth headers, timeout=10s
    - Extract last node output using: `data.resultData.runData[<last_node>][0].data.main[0]` -> list of `{json: {...}}` items
    - Return JSON array of the extracted items
    - If execution not finished or no data, return appropriate error

    Use `import json` for output formatting. Use `import logging; logger = logging.getLogger("agent42.tools.n8n_workflow")`.

    Rate limiting implementation: Use a simple module-level sliding window tracker (list of timestamps). Before each trigger call, check if len(calls in last 60s) >= 10. If so, return ToolResult(success=False, error="Rate limit exceeded: max 10 trigger calls per minute"). This is simpler than wiring into the full rate_limiter.py system since the tool manages its own limit per D-22.
