  <name>Task 2: Implement core/user_frood_dir.py to pass tests</name>
  <files>core/user_frood_dir.py</files>
  <action>
    Create `core/user_frood_dir.py`. This is bootstrap code — synchronous pathlib is acceptable (per
    CLAUDE.md, async is for tool I/O, not for startup file reads; precedent: core/key_store.py).

    Module contract (these are the ONLY public names):

    ```python
    DEFAULT_MANIFEST: dict  # module-level constant
    def user_frood_dir(create: bool = False) -> Path: ...
    def load_manifest() -> dict: ...
    def save_manifest(manifest: dict) -> None: ...
    ```

    Implementation requirements:

    1. `DEFAULT_MANIFEST` — literal dict matching the locked shape in CONTEXT.md decisions:
       ```python
       DEFAULT_MANIFEST = {
           "clis": {
               "claude-code": {"enabled": True},
               "opencode": {"enabled": True, "projects": "auto"},
           },
           "warehouse": {
               "include_claude_warehouse": True,
               "include_frood_builtins": True,
           },
       }
       ```

    2. `user_frood_dir(create=False)` — returns `Path.home() / ".frood"`. If `create=True`, calls
       `.mkdir(parents=True, exist_ok=True)` on the returned path. Never crashes if the dir
       already exists.

    3. `load_manifest()`:
       - Path = `user_frood_dir(create=True) / "cli.yaml"`.
       - If file missing: write DEFAULT_MANIFEST via `save_manifest(DEFAULT_MANIFEST)`, then return
         a deep copy of DEFAULT_MANIFEST.
       - If file present: parse. On parse error, log `logger.warning("Malformed cli.yaml: %s", e)`
         and return a deep copy of DEFAULT_MANIFEST (DO NOT overwrite the malformed file — user
         may have a typo they want to fix).
       - Deep-merge user values over a fresh copy of DEFAULT_MANIFEST so missing keys are filled.
         Implement `_deep_merge(base: dict, override: dict) -> dict` as a private helper.

    4. `save_manifest(manifest)`:
       - Ensures `user_frood_dir(create=True)` first.
       - Serializes to `cli.yaml`.
       - Serializer preference: if `import yaml` succeeds (PyYAML), use `yaml.safe_dump(manifest,
         sort_keys=False, default_flow_style=False)`. Else fall back to `json.dumps(manifest, indent=2)`
         — per Claude's Discretion in CONTEXT.md, JSON is acceptable if PyYAML isn't a dep. Log which
         backend was used at DEBUG level.
       - Use `Path.write_text(..., encoding="utf-8")` — simple, sync, atomic enough for this use.

    5. Logger: `logger = logging.getLogger("frood.user_frood_dir")`.

    6. NEVER import any other frood module except stdlib + optional `yaml`. This is bootstrap —
       must have zero circular-dep risk with `core.config`, `core.key_store`, etc.

    Per D-03 / CLI-03 (locked): absent file must not crash; partial file must fill defaults.
  </action>
