    ```
    #!/usr/bin/env python3
    # hook_event: UserPromptSubmit
    # hook_timeout: 10
    ```

    **Constants:**
    - `MIN_PROMPT_LEN = 15`
    - `MAX_OUTPUT_CHARS = 2000`
    - `DASHBOARD_URL` from `os.environ.get("AGENT42_DASHBOARD_URL", "http://127.0.0.1:8000")`
    - `INJECTION_GUARD_DIR` from `os.environ.get("AGENT42_DATA_DIR", ".agent42")`

    **TASK_TYPE_KEYWORDS dict** mapping keywords to TaskType values:
    ```python
    TASK_TYPE_KEYWORDS = {
        "coding": ["implement", "create", "build", "add", "endpoint", "api", "function", "class", "module", "component", "route", "handler", "middleware", "schema", "model", "migration"],
        "debugging": ["fix", "bug", "debug", "error", "crash", "broken", "failing", "issue", "wrong", "trace", "stack", "exception", "diagnose", "investigate"],
        "research": ["research", "compare", "evaluate", "analyze", "explore", "study", "review", "assess", "benchmark"],
        "content": ["write", "document", "readme", "docs", "blog", "content", "copy", "text", "article"],
        "strategy": ["plan", "design", "architect", "roadmap", "strategy", "structure", "organize", "refactor"],
        "app_create": ["flask app", "django app", "react app", "vue app", "fastapi app", "scaffold", "boilerplate", "starter", "new project", "new app"],
        "marketing": ["marketing", "seo", "campaign", "audience", "brand", "promotion"],
    }
    ```

    **`infer_task_type(prompt: str) -> str`:**
    - If prompt is empty or len < MIN_PROMPT_LEN, return ""
    - Lowercase the prompt
    - Check `app_create` keywords first (multi-word phrases take priority)
    - Then check other types, counting keyword matches per type
    - Return the type with the most keyword matches, or "" if no matches
    - If tie, prefer coding > debugging > strategy > research > content > marketing > app_create

    **`is_injection_done(project_dir: str, session_id: str) -> bool`:**
    - Check for `{INJECTION_GUARD_DIR}/injection-done.json` relative to project_dir
    - If file exists, read JSON, check if `session_id` matches
    - Return True if session_id matches (already injected this session)
    - Return False otherwise (file missing, different session, parse error)

    **`mark_injection_done(project_dir: str, session_id: str) -> None`:**
    - Write `{"session_id": session_id, "ts": time.time()}` to `{INJECTION_GUARD_DIR}/injection-done.json`
    - Wrapped in try/except (non-critical)

    **`fetch_learnings(query: str, task_type: str) -> list[dict]`:**
    - HTTP GET to `{DASHBOARD_URL}/api/learnings/retrieve?task_type={task_type}&query={url_encoded_query}&top_k=3&min_score=0.80`
    - Parse JSON response, return `results` list
    - Return [] on any exception (timeout=5s)

    **`format_injection_output(results: list[dict], task_type: str) -> str`:**
    - Header: `[agent42-learnings] Injecting {len(results)} past learnings for task_type={task_type}`
    - For each result: `  - [{score:.0%}] {text}`
    - Footer: `(Proactive injection — these are past learnings relevant to this task type)`
    - Truncate total output to MAX_OUTPUT_CHARS

    **`main():`**
    - Read JSON from stdin
    - Extract `user_prompt`, `project_dir`, `session_id` (from event or generate hash of project_dir as fallback session identifier)
    - Skip if prompt starts with "/" or len < MIN_PROMPT_LEN
    - Extract session_id: use `event.get("session_id", "")`. If empty, derive from project_dir hash as stable session identifier
    - Check `is_injection_done(project_dir, session_id)` — if True, exit 0 (already injected this session)
    - Call `infer_task_type(prompt)` — if empty string, exit 0 (no task type detected)
    - Call `fetch_learnings(prompt[:500], task_type)` — pass first 500 chars of prompt as query
    - If results empty, exit 0
    - Call `format_injection_output(results, task_type)`
    - Print output to stderr
    - Call `mark_injection_done(project_dir, session_id)`
    - Exit 0

    Important implementation notes:
    - Always exit 0 (never block the prompt)
    - All file/network operations wrapped in try/except
    - Use `urllib.request` for HTTP calls (same pattern as memory-recall.py, no external deps needed in hook subprocess)
    - Use `urllib.parse.urlencode` for query parameters
