**3. Replace `handleProceedToCheckout()` stub** (currently at line ~942) with the real implementation:

```javascript
async function handleProceedToCheckout() {
  if (!state.latestDesign || !state.currentProduct) {
    alert('Please select a product and create a design first.');
    return;
  }
  hideMockupModal();

  // Show loading state
  const btn = document.querySelector('.mockup-checkout-btn');
  if (btn) {
    btn.disabled = true;
    btn.textContent = 'Redirecting to checkout...';
  }

  try {
    const data = await authFetch('/api/checkout/purchase', {
      method: 'POST',
      body: JSON.stringify({
        design_id: state.latestDesign.id,
        product_id: state.currentProduct.id,
        size: state.selectedSize || 'M',
        success_url: window.location.origin + '/?section=orders',
        cancel_url: window.location.origin + '/?section=design-studio',
      }),
    });

    if (data.checkout_url) {
      window.location.href = data.checkout_url;
    } else {
      throw new Error('No checkout URL returned');
    }
  } catch (err) {
    alert('Checkout failed: ' + (err.message || 'Unknown error'));
    if (btn) {
      btn.disabled = false;
      btn.textContent = 'Proceed to Checkout';
    }
  }
}
```

Key details:
- Uses `authFetch()` (NOT raw `fetch()`) because the purchase endpoint requires authentication
- `state.latestDesign.id` is the design_id (set when AI generates or user uploads a design)
- `state.currentProduct.id` is the product_id (set when user picks a product in the design studio)
- `state.selectedSize` defaults to 'M' if not explicitly set
- `success_url` redirects to the orders section after payment
- `cancel_url` returns to the design studio
- Button is disabled during redirect to prevent double-clicks
