Auth API (from Plan 02):
```
POST /api/auth/register  -- body: {"email", "password", "name"} -- 201 {"id", "email", "name"}
POST /api/auth/login     -- body: {"email", "password"} -- 200 {"access_token", "token_type": "bearer"}

```

Catalog API (from Plan 03):
```
GET /api/catalog/products              -- 200 {"products": [{"id", "name", "category", "thumbnail_url", "price_min", "price_max"}]}

GET /api/catalog/products/{id}         -- 200 {"id", "name", "description", "category", "thumbnail_url", "variants": [{"size", "color", "color_hex", "retail_price", "in_stock"}], "images": [{"url", "position"}]}
```

    **frontend/app.js** -- Client-side SPA logic:

    State management:
    ```javascript
    const state = {
      token: localStorage.getItem('mg_token') || null,
      user: null,
      products: [],
      currentProduct: null,
      authMode: 'login', // 'login' or 'register'
    };
    ```

    Core functions to implement:

    1. `init()` -- Called on DOMContentLoaded. If `state.token` exists, call `checkAuth()` to verify. Then call `fetchProducts()`.

    2. `checkAuth()` -- `fetch('/api/auth/me', {headers: {'Authorization': 'Bearer ' + state.token}})`. On 200: set `state.user`, update nav to show user email + "Sign Out". On 401: clear token from localStorage, reset state.

    3. `fetchProducts(category = null)` -- `fetch('/api/catalog/products' + (category ? '?category=' + category : ''))`. Populate `state.products`. Call `renderProductGrid()`.

    4. `renderProductGrid()` -- For each product in `state.products`, create a card div with: product-card class, onclick showProductDetail(product.id), img with thumbnail_url and lazy loading, card-body div with h3 product name, p.price showing price range ($min - $max or just $min if same), button.btn-primary "DESIGN IT". Handle empty state: show products-empty div if no products.

    5. `showProductDetail(productId)` -- `fetch('/api/catalog/products/' + productId)`. Hide grid, show detail section. Render: image gallery (large main image + thumbnail strip, clicking thumbnail swaps main), product name (h1 Oswald), price display (selected variant retail_price), color swatches (circle divs with background-color from color_hex, click selects color), size selector (button group S/M/L/XL/2XL, disabled if not in_stock), size guide link, "DESIGN IT" CTA, description text.

    6. `showAuthModal()` / `hideAuthModal()` -- Toggle auth modal visibility.

    7. `toggleAuthMode()` -- Switch between login and register. Show/hide name field. Update button text and title.

    8. `handleAuth(event)` -- Prevent default. Register mode: POST /api/auth/register then auto-login. Login mode: POST /api/auth/login. On success: localStorage.setItem('mg_token', data.access_token), checkAuth(), hide modal. On error: show in auth-error div.

    9. `signOut()` -- Remove mg_token from localStorage, clear state.user, update nav.

    10. `showProducts()` -- Hide detail, show grid, scroll to top.

    11. `showSizeGuide()` / `hideSizeGuide()` -- Toggle size guide modal.

    All fetch calls use async/await. Error handling: show user-friendly messages, never expose raw API errors. JWT token passed via Authorization: Bearer header on authenticated requests.

    1. Add a root route `GET /` that serves `frontend/index.html` directly using `FileResponse`:
       ```python
       from fastapi.responses import FileResponse
       from pathlib import Path

       FRONTEND_DIR = Path(__file__).parent / "frontend"

       @app.get("/")
       async def root():
           return FileResponse(FRONTEND_DIR / "index.html")
       ```

    2. Ensure the `StaticFiles` mount for `/static` is AFTER all API route registrations (FastAPI matches routes in order, StaticFiles catch-all must be last):
       ```python
       # Register API routers first
       app.include_router(auth_router, prefix="/api/auth", tags=["auth"])
       app.include_router(catalog_router, prefix="/api/catalog", tags=["catalog"])

       # Static files last
       app.mount("/static", StaticFiles(directory=str(FRONTEND_DIR)), name="static")
       ```

    3. Add CORS middleware for development (same-origin in production, but useful during dev):
       ```python
       from fastapi.middleware.cors import CORSMiddleware
       app.add_middleware(
           CORSMiddleware,
           allow_origins=["http://localhost:8001", "http://127.0.0.1:8001"],
           allow_credentials=True,
           allow_methods=["*"],
           allow_headers=["*"],
       )
       ```
