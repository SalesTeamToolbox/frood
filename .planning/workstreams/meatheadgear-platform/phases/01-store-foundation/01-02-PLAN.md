    **services/auth.py** — Auth utility functions:

    ```python
    from datetime import datetime, timedelta, timezone
    from jose import jwt, JWTError
    from passlib.context import CryptContext
    import secrets

    pwd_context = CryptContext(schemes=["bcrypt"], deprecated="auto")

    def hash_password(password: str) -> str:
        return pwd_context.hash(password)

    def verify_password(plain: str, hashed: str) -> bool:
        return pwd_context.verify(plain, hashed)

    def create_access_token(user_id: int, email: str, secret_key: str, expiry_days: int = 7) -> str:
        expire = datetime.now(timezone.utc) + timedelta(days=expiry_days)
        payload = {"sub": str(user_id), "email": email, "exp": expire}
        return jwt.encode(payload, secret_key, algorithm="HS256")

    def verify_access_token(token: str, secret_key: str) -> dict | None:
        try:
            payload = jwt.decode(token, secret_key, algorithms=["HS256"])
            return payload
        except JWTError:
            return None

    def generate_reset_token() -> str:
        return secrets.token_urlsafe(32)
    ```

    **routers/auth.py** — FastAPI router with 5 endpoints:

    - `POST /register` — Accept `{"email": str, "password": str, "name": str = ""}`. Validate password >= 8 chars. Hash password with bcrypt. INSERT into users table. Return 201 with `{"id", "email", "name"}`. Return 409 if email exists (catch UNIQUE constraint).
    - `POST /login` — Accept `{"email": str, "password": str}`. SELECT user by email. Verify password with bcrypt. Return 200 with `{"access_token": jwt, "token_type": "bearer"}`. Return 401 on invalid credentials (same message for wrong email and wrong password — no enumeration).
    - `GET /me` — Extract Bearer token from `Authorization` header. Decode JWT. SELECT user by id from payload `sub`. Return 200 with `{"id", "email", "name", "created_at"}`. Return 401 if no token, invalid token, or expired token.
    - `POST /reset-request` — Accept `{"email": str}`. If user exists, generate reset token via `generate_reset_token()`, set `reset_token` and `reset_token_expires` (1 hour from now) on user row. Always return 200 `{"message": "If an account exists, a reset link has been sent"}` (no email enumeration). NOTE: Actual email sending via Resend is a stub for now (log the token) — Resend API key may not be available yet.
    - `POST /reset-confirm` — Accept `{"token": str, "new_password": str}`. SELECT user by reset_token. Check reset_token_expires > now. Hash new password. UPDATE user password_hash and clear reset_token fields. Return 200 on success, 400 on invalid/expired token.

    Use FastAPI `Depends` for a `get_current_user` dependency:
    ```python
    from fastapi import Depends, HTTPException, status
    from fastapi.security import HTTPBearer, HTTPAuthorizationCredentials

    security = HTTPBearer()

    async def get_current_user(
        credentials: HTTPAuthorizationCredentials = Depends(security),
        db = Depends(get_db)
    ) -> dict:
        payload = verify_access_token(credentials.credentials, settings.secret_key)
        if payload is None:
            raise HTTPException(status_code=401, detail="Invalid or expired token")
        cursor = await db.execute("SELECT id, email, name, created_at FROM users WHERE id = ?", (int(payload["sub"]),))
        user = await cursor.fetchone()
        if user is None:
            raise HTTPException(status_code=401, detail="User not found")
        return {"id": user[0], "email": user[1], "name": user[2], "created_at": user[3]}
    ```
