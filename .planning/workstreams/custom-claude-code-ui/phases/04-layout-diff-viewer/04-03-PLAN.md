**Part A: Replace the stub `ideToggleCCPanel()` with the full implementation**

Find the existing stub function `function ideToggleCCPanel()` (added in Plan 04-02) and replace it entirely with these functions:

1. `ideToggleCCPanel()` - Three-state toggle:
   - If `_ccPanelMode` is true: call `ideMoveSessionsToTab()` then `ideCloseCCPanel()`
   - Else if there are CC tabs in `_ideTabs` (filter by `t.type === "claude"`): call `ideOpenCCPanel()` then `ideMoveSessionsToPanel()`
   - Else (no CC open): call `ideOpenCCPanel()`, then `ideOpenCCChat("local")`, then `setTimeout(ideMoveSessionsToPanel, 50)` to move the newly-created tab into the panel

2. `ideOpenCCPanel()` - Shows the panel:
   - Get `#ide-cc-panel` and `#ide-panel-drag-handle` elements
   - Read saved width from `localStorage.getItem("cc_panel_width")`, default 400
   - Set panel `display: "flex"`, `flexDirection: "column"`, `width: savedWidth + "px"`, `flex: "none"`
   - Show drag handle (`display: ""`)
   - Set `_ccPanelMode = true`
   - Call `_monacoEditor.layout()` in a 50ms setTimeout

3. `ideCloseCCPanel()` - Hides the panel:
   - Persist width to localStorage via `panel.getBoundingClientRect().width`
   - Set panel and handle `display: "none"`
   - Set `_ccPanelMode = false`
   - Call `_monacoEditor.layout()` in a 50ms setTimeout

4. `ideMoveSessionsToPanel()` - Transfers CC tab DOM elements to panel:
   - Get `#ide-cc-panel` element
   - For each CC tab in `_ideTabs` (type === "claude"): call `panel.appendChild(tab.el)` to MOVE (not clone) the DOM element; set `tab.el.style.display = tab.chatPanel ? "flex" : "block"`; set `tab.inPanel = true`
   - Hide `#ide-cc-container` (display: "none")
   - Find first non-claude tab and activate it (show editor); if none, show `#ide-welcome`
   - Call `ideRenderTabs()` to refresh tab bar

5. `ideMoveSessionsToTab()` - Transfers CC tab DOM elements back:
   - Get `#ide-cc-container` element
   - For each CC tab: call `ccContainer.appendChild(tab.el)` to move back; set `tab.inPanel = false`
   - Activate the first CC tab by setting `_ideActiveTab` and calling `ideActivateTab()`
   - Call `ideRenderTabs()` to refresh tab bar

CRITICAL: The move functions use `appendChild()` which moves existing DOM nodes. This preserves all event listeners and the WS connection on `tab.ws`. Do NOT call `tab.ws.close()` anywhere in the move logic.
