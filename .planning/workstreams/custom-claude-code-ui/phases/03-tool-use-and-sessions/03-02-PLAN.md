
**3b. In the `input_json_delta` branch, accumulate partials into `tool_id_map[index]["input_buf"]`:**

```python
elif raw_type == "input_json_delta":
    partial = event.get("delta", {}).get("partial_json", "")
    if index in tool_id_map:
        tool_id_map[index]["input_buf"] = tool_id_map[index].get("input_buf", "") + partial
        if not tool_id_map[index].get("is_permission"):
            envelopes.append(
```

**4. In the `content_block_stop` branch (~line 1966), emit permission_request with populated input OR tool_complete:**
Modify the existing content_block_stop handler. For the permission tool, parse the accumulated input buffer and emit `permission_request` with the full input. For regular tools, emit `tool_complete` as before:
```python
elif raw_type == "content_block_stop":
    if index in tool_id_map:
        t = tool_id_map.pop(index)
        if t.get("is_permission"):
            # Parse accumulated input — now has the full tool input JSON
            perm_input = {}
            try:
                import json as _json_mod
                perm_input = _json_mod.loads(t.get("input_buf", "{}"))
            except Exception:
                pass
            envelopes.append(
                {
                    "type": "permission_request",
                    "data": {"id": t["id"], "name": t["name"], "input": perm_input},
                }
            )
        else:
            envelopes.append(
                {
                    "type": "tool_complete",
                    "data": {
                        "id": t.get("id"),
                        "name": t.get("name"),
                        "output": "",
                        "is_error": False,
                    },
                }
            )
```

**Why this ordering matters (Blocker fix):** CC streams `content_block_start` (tool name/id only), then N `input_json_delta` events (partial JSON chunks), then `content_block_stop`. The original plan emitted `permission_request` on `content_block_start` before any input arrived, so the frontend received `input: {}` — making the permission card unable to show what tool CC wants to use. By deferring `permission_request` to `content_block_stop`, the full parsed input (tool_name, file_path, command, etc.) is included.

These changes add ~40 lines net to `_parse_cc_event` (more than original estimate due to input buffering for permission tool).
