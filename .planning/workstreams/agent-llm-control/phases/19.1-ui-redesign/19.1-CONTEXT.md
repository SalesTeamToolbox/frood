# Phase 19.1: UI Redesign - Context

**Gathered:** 2026-03-16
**Status:** Ready for planning

<domain>
## Phase Boundary

Redesign the coding/IDE page to mirror VS Code's CC/terminal integration. Fix layout (no scroll-to-find), establish clear local vs remote distinction, fix broken remote connections, and implement proper panel layout (editor top, terminal/CC bottom). Remove the AI Chat side panel. This phase does NOT touch other dashboard pages (Settings, Agents, Chat, etc.).

</domain>

<decisions>
## Implementation Decisions

### Panel Layout
- VS Code style: editor fills top, resizable bottom panel for terminal/CC, status bar at bottom
- No side chat panel — the AI Chat panel (right side, Anthropic API direct) is removed entirely
- Bottom panel visible by default on page load with a default local terminal tab
- Draggable divider between editor and terminal panel — terminal starts at ~30% height
- Toggle via status bar button + Ctrl+` keyboard shortcut (matches VS Code)
- Panel minimizes/restores (not destroyed) when toggled

### Claude Code Integration
- Terminal-based CC is the only AI experience — CC runs as a real CLI session in the bottom panel
- CC sessions appear as regular tabs alongside terminal tabs (same tab bar)
- Tab label shows type: "Claude (local)" or "Claude (remote)"
- AI Chat panel code (ideChatToggle, ideChatSend, ideChatHistory, etc.) is removed completely

### Local vs Remote Clarity
- Single "+" button opens a dropdown menu grouped into "Local" (Terminal, Claude Code) and "Remote" (Terminal, Claude Code)
- Matches VS Code's terminal profile dropdown pattern
- Color-coded dot on each tab: blue for local, green for remote
- Tab labels: "● bash 1", "● Claude (local)", "● bash (remote)", "● Claude (remote)"
- Remote options grayed out with tooltip ("No remote node configured") when no remote node available
- Remote node availability checked at dropdown render time

### Terminal Reliability
- Fix the remote terminal/CC WebSocket → SSH pipeline — both remote terminal and remote CC must work
- Auto-fit xterm.js on resize: FitAddon recalculates when divider is dragged or window resizes, sends SIGWINCH to remote shell
- Auto-reconnect on WebSocket drop: show "Disconnected — reconnecting..." in terminal, auto-retry with backoff (1s, 2s, 4s), tab shows warning dot, reconnects to new session

### Claude's Discretion
- Exact CSS for the draggable divider (cursor, hover state, width)
- Dropdown menu styling and positioning
- Status bar layout and content
- How to detect remote node availability (ping endpoint vs config check)
- Auto-reconnect max retry count and final failure behavior

</decisions>
