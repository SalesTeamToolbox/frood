## Summary

Phase 19.1 is a surgical UI redesign of the IDE/coding page in `dashboard/frontend/dist/app.js` and `style.css`. The dashboard is a vanilla JS SPA (~5082 lines) with no build step — changes are made directly to the dist files. The existing xterm.js and FitAddon are already bundled locally (served from `/xterm/`), so no new library installs are needed.

The core work is: (1) restructure `renderCode()` HTML from the current flat layout to a flex-column layout with a draggable divider, (2) replace the four-button terminal header with a single "+" dropdown menu, (3) add color-coded local/remote dots to tabs, (4) remove all `ideChat*` code, (5) fix the remote SSH pipeline and add auto-reconnect logic with exponential backoff, and (6) auto-fit xterm.js when the divider is dragged.

The remote connection bug — local Claude unexpectedly opening a remote VPS connection — is caused by logic in `server.py`'s `/ws/terminal` endpoint where `AGENT42_REMOTE_HOST` defaults to `agent42-prod` for the `cmd=claude&node=remote` path. The frontend already sends `node=local` for local Claude sessions, so the issue is the backend defaulting to the SSH host even when not explicitly configured. A second bug path: if anything sends `node=remote` unintentionally, remote SSH fires.

**Primary recommendation:** Modify `renderCode()` to use a vertical flex layout with a drag handle, implement the "+" dropdown with grouped local/remote options, add color dots to `_termSessions` metadata, delete all `ideChat*` code, and add a `termReconnect()` function with exponential backoff. Backend `/ws/terminal` needs JSON resize message parsing.

### Pattern 1: Vertical Drag Handle (Resizable Panel)

**What:** A 4px div between editor area and terminal panel. mousedown on handle, track mousemove on document, update terminal panel height on drag.
**When to use:** This is the standard VS Code sash pattern for panel resizing.

```javascript
// Drag handle implementation
var _isDragging = false;
var _dragStartY = 0;
var _dragStartHeight = 0;

function initDragHandle() {
  var handle = document.getElementById("ide-drag-handle");
  if (!handle) return;
  handle.addEventListener("mousedown", function(e) {
    _isDragging = true;
    _dragStartY = e.clientY;
    var wrapper = document.getElementById("ide-terminal-wrapper");
    _dragStartHeight = wrapper ? wrapper.getBoundingClientRect().height : 200;
    document.body.style.cursor = "ns-resize";
    document.body.style.userSelect = "none";
    e.preventDefault();
  });
  document.addEventListener("mousemove", function(e) {
    if (!_isDragging) return;
    var delta = _dragStartY - e.clientY; // drag up = increase panel height
    var newHeight = Math.max(80, Math.min(_dragStartHeight + delta, window.innerHeight * 0.8));
    var wrapper = document.getElementById("ide-terminal-wrapper");
    if (wrapper) {
      wrapper.style.height = newHeight + "px";
      wrapper.style.flex = "none"; // override default flex sizing
    }
    termFitAll(); // refit all terminals after drag
  });
  document.addEventListener("mouseup", function() {
    if (!_isDragging) return;
    _isDragging = false;
    document.body.style.cursor = "";
    document.body.style.userSelect = "";
  });
}
```

**CRITICAL:** Store `fitAddon` on each session object — currently it is a local variable in `termNew()` and lost after the function returns.

### Pattern 3: Color-Coded Tabs

**What:** Add node color dot as Unicode circle before tab label. Use amber for reconnecting state.
**When to use:** Every tab render via `termRenderTabs()`.

```javascript
function termRenderTabs() {
  var el = document.getElementById("ide-terminal-tabs");
  if (!el) return;
  var html = "";
  for (var i = 0; i < _termSessions.length; i++) {
    var s = _termSessions[i];
    var active = i === _termActiveIdx ? " active" : "";
    var dotColor = s.reconnecting ? "#f59e0b" : (s.node === "remote" ? "#34d399" : "#38bdf8");
    // Use esc() for XSS safety on all interpolated values (existing project pattern)
    html += '<span class="ide-terminal-tab' + active + '" onclick="termSwitch(' + i + ')">'
      + '<span style="color:' + dotColor + ';margin-right:4px">&#9679;</span>'
      + esc(s.label)
      + ' <span class="close" onclick="event.stopPropagation();termClose(' + i + ')">&times;</span>'
      + '</span>';
  }
  el.innerHTML = html; // safe: esc() applied to all dynamic content
}
```

### Pattern 4: Auto-Reconnect with Exponential Backoff

**What:** On `ws.onclose`, retry with 1s, 2s, 4s delays. After 3 failures show permanent error in terminal.
**When to use:** All WebSocket terminal sessions (local and remote).

```javascript
function termConnectWs(session, wsUrl, retryCount) {
  retryCount = retryCount || 0;
  var ws = new WebSocket(wsUrl);
  session.ws = ws;
  session.reconnecting = retryCount > 0;
  termRenderTabs(); // update dot color immediately

  ws.onopen = function() {
    session.reconnecting = false;
    session.retryCount = 0;
    termRenderTabs();
    if (retryCount > 0) {
      session.term.write("\r\n\x1b[32mReconnected\x1b[0m\r\n");
    }
  };
  ws.onmessage = function(e) { session.term.write(e.data); };
  ws.onclose = function() {
    if (retryCount >= 3) {
      session.term.write("\r\n\x1b[31mDisconnected (max retries reached)\x1b[0m\r\n");
      session.reconnecting = false;
      termRenderTabs();
      return;
    }
    var delay = Math.pow(2, retryCount) * 1000; // 1000, 2000, 4000 ms
    session.term.write("\r\n\x1b[33mDisconnected — reconnecting in " + (delay / 1000) + "s...\x1b[0m\r\n");
    session.reconnecting = true;
    termRenderTabs();
    setTimeout(function() {
      if (_termSessions.indexOf(session) !== -1) { // session still active
        termConnectWs(session, wsUrl, retryCount + 1);
      }
    }, delay);
  };
  session.term.onData(function(data) {
    if (ws.readyState === 1) ws.send(data);
  });
}
```

### Pattern 5: Resize Message Protocol (Best-Effort)

**What:** After `fitAddon.fit()`, send terminal dimensions to backend as JSON over the same WebSocket. Backend parses and attempts to signal the subprocess.
**When to use:** After every fit call (drag, window resize, tab switch).

Frontend — send dimensions after fit:
```javascript
function termFitAll() {
  _termSessions.forEach(function(s) {
    if (!s.fitAddon) return;
    try {
      s.fitAddon.fit();
      if (s.ws && s.ws.readyState === 1 && s.term) {
        s.ws.send(JSON.stringify({ type: "resize", cols: s.term.cols, rows: s.term.rows }));
      }
    } catch (e) {}
  });
}
```

Backend — detect JSON resize messages before forwarding to stdin:
```python
# In the receive loop inside terminal_ws()
msg = await websocket.receive_text()
try:
    parsed = json.loads(msg)
    if parsed.get("type") == "resize":
        cols = int(parsed.get("cols", 80))
        rows = int(parsed.get("rows", 24))
        # Best-effort: send ANSI resize escape to process stdin
        # Works for some terminal programs; does not trigger true SIGWINCH without PTY
        if proc.stdin and not proc.stdin.is_closing():
            proc.stdin.write(f"\x1b[8;{rows};{cols}t".encode())
            await proc.stdin.drain()
        continue  # Do not forward resize message as terminal input
except (json.JSONDecodeError, ValueError, KeyError):
    pass  # Not a resize message — fall through to normal input handling
# Normal input:
if proc.stdin and not proc.stdin.is_closing():
    proc.stdin.write(msg.encode("utf-8"))
    await proc.stdin.drain()
```

### Pattern 6: Remote Node Availability Check

**What:** Lightweight API endpoint returns whether AGENT42_REMOTE_HOST is configured. Cached on frontend for 30s.
**Recommendation:** Config check (not live SSH ping) to avoid latency on dropdown open.

Backend endpoint to add:
```python
@app.get("/api/remote/status")
async def remote_status(_user: str = Depends(get_current_user)):
    import os as _os2
    host = _os2.environ.get("AGENT42_REMOTE_HOST", "")
    return {"available": bool(host), "host": host if host else None}
```

Frontend cached check:
```javascript
var _remoteAvailCache = null;
var _remoteAvailCacheAt = 0;

function checkRemoteAvailable(cb) {
  var now = Date.now();
  if (_remoteAvailCache !== null && now - _remoteAvailCacheAt < 30000) {
    cb(_remoteAvailCache);
    return;
  }
  apiFetch("/api/remote/status")
    .then(function(d) {
      _remoteAvailCache = !!(d && d.available);
      _remoteAvailCacheAt = Date.now();
      cb(_remoteAvailCache);
    })
    .catch(function() { cb(false); });
}
```

### Anti-Patterns to Avoid

- **Destroying terminal on toggle:** Never call `term.dispose()` or `el.remove()` when toggling the panel. Set `display:none` only. Disposing xterm.js kills the WebSocket and loses scroll history.
- **Multiple window.resize listeners:** The current code adds a new listener on every `termNew()` call. Fix: one shared `termFitAll` listener added once in `renderCode()`.
- **Floating fitAddon references:** Current `termNew()` keeps fitAddon in a closure but not on the session object. After tab switch, the inactive terminal cannot be refitted. Fix: `session.fitAddon = fitAddon` immediately after `term.loadAddon(fitAddon)`.
- **AMD/Monaco conflict:** `termLoadXterm()` already handles this with `savedDefine`/`savedRequire`. Do not modify this pattern.
- **Dropdown opening downward into viewport edge:** Since the "+" button is in the terminal header at the bottom of the screen, position the dropdown `bottom:100%` (opens upward) to avoid clipping by the status bar.

## Common Pitfalls

### Pitfall 1: fitAddon Lost in Closure
**What goes wrong:** `termNew()` creates `fitAddon` as a local variable. After the function returns, only the `window.resize` closure holds a reference. The drag handler cannot refit inactive tabs.
**Why it happens:** The current code never persists fitAddon on the session object.
**How to avoid:** `session.fitAddon = fitAddon` immediately after `term.loadAddon(fitAddon)`. Drag handler calls `_termSessions.forEach(s => s.fitAddon && s.fitAddon.fit())`.
**Warning signs:** Terminal looks correct on load but mismatches size after panel resize or tab switch.

### Pitfall 2: Multiple window.resize Listeners Accumulating
**What goes wrong:** Every `termNew()` call adds another `window.addEventListener("resize", ...)`. Five tabs open = five listeners on every window resize.
**Why it happens:** Current code registers per-session listeners without tracking or cleanup.
**How to avoid:** Replace per-session listener with one `window.addEventListener("resize", termFitAll)` registered once during `renderCode()`. Gate with `window._ideResizeListenerAttached`.
**Warning signs:** Performance degrades with many terminal tabs; memory leak on long sessions.

### Pitfall 3: Terminal Panel DOM Removal vs Hide
**What goes wrong:** Calling `.remove()` on `#ide-terminal-wrapper` instead of setting `display:none` destroys the xterm.js mount point. Re-initialization loses scroll history and requires new WebSocket connections.
**Why it happens:** "Toggling" a panel intuitively suggests removal.
**How to avoid:** Always use `display:none` / `display:flex` toggling. The existing `termToggle()` does this correctly — preserve it.
**Warning signs:** Terminal is blank or loses history after toggle.

### Pitfall 4: Drag Stops Tracking Cursor Mid-Drag
**What goes wrong:** `mousemove` listener on the handle element stops firing when the cursor moves faster than DOM updates (cursor leaves the element).
**Why it happens:** Events are scoped to the element.
**How to avoid:** Attach `mousedown` to the handle element, but `mousemove` and `mouseup` to `document`. Standard drag implementation pattern.
**Warning signs:** Panel resize stops tracking cursor when dragged quickly.

### Pitfall 5: Remote "Local" Bug — Local Claude Opening Remote Session
**What goes wrong:** A local Claude Code session unexpectedly spawns SSH connection to `agent42-prod`.
**Root cause (confirmed):** Backend `terminal_ws` has this logic:
```
if cmd == "claude" and node == "remote": → ssh agent42-prod claude
elif cmd == "claude": → local claude binary
elif node == "remote": → ssh agent42-prod shell
else: → local shell
```
Frontend `termNewClaude("local")` sends `node=local&cmd=claude` which correctly hits the `elif cmd == "claude"` branch (local). However, if `AGENT42_REMOTE_HOST` is not set (empty env var), `os.environ.get("AGENT42_REMOTE_HOST", "agent42-prod")` still defaults to `agent42-prod`. This means remote sessions always target `agent42-prod` even if the user has no SSH config entry for that name.
**How to avoid:** Backend should return an error when remote is requested but `AGENT42_REMOTE_HOST` is not configured (empty string), rather than defaulting to a hostname. Add check: if not `ssh_host`, close WebSocket with error message.
**Warning signs:** Remote VPS CPU spikes when opening a local Claude tab; user sees "Connected" but is actually on prod.

### Pitfall 6: SIGWINCH Without PTY (Known Limitation)
**What goes wrong:** Resize escape sequences sent to subprocess stdin do not trigger shell resize because there is no PTY.
**Why it happens:** Current backend uses `asyncio.create_subprocess_exec` without PTY allocation. On a raw pipe, neither bash nor Claude CLI will respond to `\x1b[8;rows;cols]t`.
**How to avoid:** Implement the best-effort resize message protocol (JSON over WebSocket → escape to stdin). Document this as a known limitation. Full PTY support requires the `pty` module (Linux/macOS only) and is a future phase concern.
**Warning signs:** `stty size` inside bash shows wrong dimensions; Claude CLI layout wraps at wrong column width.

### Pitfall 7: Dropdown Opens Downward Into Status Bar
**What goes wrong:** The "+" button is near the bottom of the screen (terminal header). A `top:100%` dropdown extends into the status bar or off-screen.
**Why it happens:** Standard dropdown positioning assumes the trigger is near the top.
**How to avoid:** Use `bottom:100%` (opens upward from button) for terminal header dropdowns.
**Warning signs:** Dropdown clipped or overlaps status bar.

### Pitfall 8: Ctrl+` Listener Duplicates on Page Revisit
**What goes wrong:** `renderCode()` is called on every navigation to the Code page. Without a guard, keydown listeners accumulate — pressing Ctrl+` fires toggle N times.
**Why it happens:** SPA routing pattern calls `renderCode()` on each page visit.
**How to avoid:** Guard with `window._ideKeyListenerAttached = true` set on first registration.
**Warning signs:** Terminal panel flickers (opens and immediately closes) on Ctrl+`.

## Open Questions

1. **PTY vs raw subprocess for resize**
   - What we know: Backend uses `asyncio.create_subprocess_exec` without PTY. Sending escape sequences to raw stdin does not reliably trigger SIGWINCH in the subprocess.
   - What's unclear: Whether the Claude CLI TUI responds to `\x1b[8;rows;cols;t` on a raw pipe. Whether bash does.
   - Recommendation: Implement best-effort resize message protocol. Document as known limitation. Full PTY support (requires `pty` module, Linux/macOS only) is a future phase.

2. **Remote node availability — default host**
   - What we know: `AGENT42_REMOTE_HOST` defaults to `"agent42-prod"` in the backend code. The SSH config entry `agent42-prod` may or may not exist on the user's machine.
   - What's unclear: Whether `agent42-prod` is always a valid SSH alias on developer machines (it is in production per MEMORY.md, but may not be on other dev machines).
   - Recommendation: Treat empty `AGENT42_REMOTE_HOST` env var as "no remote configured" and gray out remote options. Backend should also error if remote requested but env var not set.

3. **Backend `/api/ide/chat` endpoint**
   - What we know: The frontend AI Chat panel is removed. The endpoint at `dashboard/server.py:1552` currently handles direct Anthropic API chat.
   - What's unclear: Whether any other code path uses this endpoint.
   - Recommendation: Leave backend endpoint in place. Only remove frontend UI. Confirm in code review that no other routes call it before any future removal.
