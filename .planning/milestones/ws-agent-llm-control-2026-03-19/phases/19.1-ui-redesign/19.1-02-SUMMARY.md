## Decisions Made

- Terminal wrapper is a direct child of ide-layout (sibling to ide-top-row), not nested inside ide-main — this is required so the drag handle spans full width
- Status bar is the last child of ide-layout, ensuring it stays pinned at the very bottom regardless of terminal height
- fitAddon stored on session object immediately after loadAddon — fixes the closure-loss bug that prevented drag-handle resize from refitting inactive tabs
- Listener deduplication guards use window-level flags (_ideKeyListenerAttached, _ideResizeListenerAttached) since renderCode() is called on every SPA navigation to the Code page
- Dropdown positioned with bottom:calc(100%+4px) (opens upward) to avoid being clipped by the status bar at screen bottom
- Backend /api/ide/chat endpoint left in place (frontend-only removal per plan scope)
