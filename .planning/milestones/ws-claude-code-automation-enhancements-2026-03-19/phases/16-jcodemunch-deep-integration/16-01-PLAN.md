  <files>.claude/hooks/jcodemunch-reindex.py, tests/test_jcodemunch_drift.py</files>
  <behavior>
    - Test 1: check_drift with tool_name "mcp__jcodemunch__get_symbol" and content_verified=false returns True
    - Test 2: check_drift with tool_name "mcp__jcodemunch__get_symbol" and content_verified=true returns False
    - Test 3: check_drift with tool_name "mcp__jcodemunch__search_symbols" (non-get_symbol) returns False regardless of content
    - Test 4: check_drift with missing _meta returns False (graceful degradation)
    - Test 5: check_drift with malformed tool_output (string, None, empty dict) returns False
    - Test 6: Full integration — simulate PostToolUse event with drifted get_symbol response, verify stderr contains re-index recommendation
  </behavior>
  <action>
    1. Modify jcodemunch-reindex.py to handle BOTH Stop events (existing) AND PostToolUse events (new drift detection).

    2. Add a check_drift(tool_name, tool_output) function that:
       - Returns False if tool_name doesn't contain "get_symbol"
       - Parses tool_output (may be dict or JSON string) to extract _meta.content_verified
       - Returns True if content_verified is explicitly False
       - Returns False for any other case (True, missing, malformed)

    3. In main(), BEFORE the Stop event check, add PostToolUse handling:
       ```python
       hook_event = event.get("hook_event_name", "")

       # Handle PostToolUse: drift detection
       if hook_event == "PostToolUse":
           tool_name = event.get("tool_name", "")
           if "jcodemunch" in tool_name:
               tool_output = event.get("tool_output", {})
               if check_drift(tool_name, tool_output):
                   file_info = ""
                   if isinstance(tool_output, dict):
                       file_info = tool_output.get("file_path", "")
                   print(
                       f"[jcodemunch] Source drift detected"
                       f"{' in ' + file_info if file_info else ''}! "
                       "Index is stale. Consider re-indexing:\n"
                       "  Call mcp__jcodemunch__index_folder with:\n"
                       f'    path: "{project_dir}"\n'
                       "    incremental: true",
                       file=sys.stderr,
                   )
           sys.exit(0)  # Advisory only, never block

       # Existing Stop event handling continues below...
       if hook_event != "Stop":
           sys.exit(0)
       ```

    4. Register jcodemunch-reindex.py as a PostToolUse hook in .claude/settings.json, in addition to its existing Stop registration. Add it to the PostToolUse array with a catch-all matcher (no specific matcher needed since the hook itself filters for jcodemunch tool names). Use timeout: 10 since drift checks are lightweight.

    IMPORTANT: Drift detection is ADVISORY (exit 0). It must NEVER block (exit 2). Only the existing Stop event structural change detection blocks.
    IMPORTANT: Only check drift on get_symbol responses — search_symbols and get_file_outline don't have verify capability.
  </action>
