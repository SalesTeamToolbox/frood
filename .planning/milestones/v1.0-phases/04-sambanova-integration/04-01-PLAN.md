<objective>
Register SambaNova as a credits-based provider with two models and implement three request transforms that prevent known SambaNova API failure modes.

Purpose: SambaNova provides OpenAI-compatible inference but deviates from the spec in three ways: (1) rejects temperature > 1.0, (2) returns malformed `index` fields in streaming tool call responses, (3) does not support `strict: true` in tool definitions. This plan registers the provider (identical pattern to Phases 1-3) AND adds inline request transforms in `complete()` and `complete_with_tools()` -- the first provider requiring parameter mutation before API call.

Output: Modified `providers/registry.py` with 1 ProviderSpec, 2 ModelSpecs (CHEAP tier), 2 non-zero pricing entries, and 3 inline request transforms. Modified `core/config.py` with `sambanova_api_key` field. Updated `.env.example` with documentation.
</objective>
