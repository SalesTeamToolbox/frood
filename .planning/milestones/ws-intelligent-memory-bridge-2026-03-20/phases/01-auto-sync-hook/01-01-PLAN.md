**File 1: `.claude/hooks/cc-memory-sync.py` (hook entry point)**

This file uses ONLY Python stdlib — zero Agent42 imports (keeps startup < 5ms).

```python
#!/usr/bin/env python3
# hook_event: PostToolUse
# hook_matcher: Write|Edit
# hook_timeout: 5
```

Implement:
1. `is_cc_memory_file(file_path: str) -> bool` — Checks if `file_path` matches `~/.claude/projects/*/memory/*.md` using `Path(file_path).parts`:
   - Find the index of `.claude` in parts
   - Verify parts[idx+1] == "projects", parts[idx+3] == "memory", and the file has .md suffix
   - Return False on any ValueError/IndexError or if not enough parts exist
   - This must work with both Windows backslash paths and Unix forward-slash paths

2. `main()`:
   - Read JSON from stdin: `event = json.loads(sys.stdin.read())`
   - Extract `tool_name` from event; exit 0 if not "Write" or "Edit"
   - Extract `file_path` from `event.get("tool_input", {}).get("file_path", "")`
   - Exit 0 if file_path is empty or `is_cc_memory_file()` returns False
   - Resolve worker path: `Path(__file__).parent / "cc-memory-sync-worker.py"`
   - Spawn detached subprocess:
     ```python
     creation_flags = 0
     if sys.platform == "win32":
         creation_flags = subprocess.DETACHED_PROCESS | subprocess.CREATE_NEW_PROCESS_GROUP
     subprocess.Popen(
         [sys.executable, str(worker), file_path],
         stdin=subprocess.DEVNULL,
         stdout=subprocess.DEVNULL,
         stderr=subprocess.DEVNULL,
         close_fds=(sys.platform != "win32"),
         creationflags=creation_flags,
     )
     ```
   - `sys.exit(0)` — always exit 0, never block CC (SYNC-04)

3. `if __name__ == "__main__": main()`

**File 2: `.claude/hooks/cc-memory-sync-worker.py` (background worker)**

This file DOES import Agent42 modules (runs as detached process, not in hook context).

Bootstrap section at top of file:
```python
import sys
from pathlib import Path
script_dir = Path(__file__).resolve().parent       # .claude/hooks/
project_dir = script_dir.parent.parent             # project root
sys.path.insert(0, str(project_dir))
```

Implement:
1. `STATUS_FILE = project_dir / ".agent42" / "cc-sync-status.json"`

2. `make_point_id(file_path: str) -> str`:
   - `content = f"claude_code:{file_path}"`
   - `namespace = uuid.UUID("a42a42a4-2a42-4a42-a42a-42a42a42a42a")`
   - Return `str(uuid.uuid5(namespace, content))`

3. `load_status() -> dict`: Read STATUS_FILE, return dict or `{"last_sync": None, "total_synced": 0, "last_error": None}`

4. `save_status(status: dict)`: Create parent dirs, write JSON to STATUS_FILE

5. `parse_frontmatter(content: str) -> dict`: Extract YAML frontmatter between `---` delimiters. Parse key-value pairs (simple string splitting, no PyYAML dependency). Return dict with keys like `name`, `description`, `type`.

6. `sync_memory_file(file_path: str)` — The main sync function:
   - Call `load_status()`
   - Read file content from `file_path` (exit if file missing or empty)
   - Call `parse_frontmatter(content)` to extract metadata
   - Call `_find_onnx_model_dir()` from `memory.embeddings`; if None, record error in status and return
   - Create `_OnnxEmbedder(model_dir)` and call `embedder.encode(content[:2000])` to get 384-dim vector
   - Wrap embedding in try/except; on failure, record error and return
   - Load Qdrant config from env vars:
     ```python
     import os
     qdrant_url = os.getenv("QDRANT_URL", "")
     qdrant_local = os.getenv("QDRANT_LOCAL_PATH", str(project_dir / ".agent42" / "qdrant"))
     config = QdrantConfig(url=qdrant_url, local_path=qdrant_local, vector_dim=384)
     ```
   - Create `QdrantStore(config)` and check `store.is_available`; if False, return silently (SYNC-04)
   - Build payload:
     ```python
     payload = {
         "source": "claude_code",
         "file_path": file_path,
         "section": Path(file_path).stem,
         **frontmatter,  # name, description, type from YAML if present
     }
     ```
   - Call `store.upsert_single(QdrantStore.MEMORY, content, vector, payload)` — uses the MEMORY collection
   - Note: point ID is generated inside `upsert_vectors` via `_make_point_id(text, payload.get("source", ""))` which produces `uuid5(namespace, "claude_code:{content}")`. However, this creates a CONTENT-based ID, not a FILE-PATH-based ID. To get file-path-based dedup (SYNC-03), we need the upsert to use the file path as the identity key. Solution: set `payload["source"] = f"cc:{file_path}"` so that `_make_point_id(text=content, source=f"cc:{file_path}")` produces `uuid5(namespace, f"cc:{file_path}:{content}")`. BUT this still includes content in the hash, meaning edits create new points.

   **CRITICAL DEDUP FIX:** The existing `_make_point_id(text, source)` uses content in the hash, which breaks file-path-based dedup. Instead, call `upsert_vectors` directly with a manually-computed point ID:
     ```python
     from qdrant_client.models import PointStruct
     point_id = make_point_id(file_path)  # deterministic UUID5 from file path only
     full_payload = {"text": content, "timestamp": time.time(), **payload}
     point = PointStruct(id=point_id, vector=vector, payload=full_payload)
     collection_name = f"{config.collection_prefix}_memory"
     store._ensure_collection(QdrantStore.MEMORY)
     store._client.upsert(collection_name=collection_name, points=[point])
     ```
   - Update status: `{"last_sync": time.time(), "total_synced": prev + 1, "last_error": None}`
   - All exceptions caught with bare `except Exception`: record error in status, return silently

7. `if __name__ == "__main__":` — call `sync_memory_file(sys.argv[1])` wrapped in try/except that silently exits

**IMPORTANT:** The worker must NEVER write to stdout or stderr. All error reporting goes to the status file only.
