# Agent42 v3.0 — Autonomous Agent Platform with Web IDE

**Vision:** Agent42 evolves from MCP server ecosystem to a full autonomous agent
platform with web IDE, custom agent creation, and 24/7 agent management.

**Three pillars:**
1. **MCP Server** (v2.0, done) — Tools for Claude Code in VS Code
2. **Web IDE** (new) — Browser-based development environment with Claude Code integration
3. **Agent Management** (new) — Create, deploy, and monitor custom autonomous agents

## Phase 1: Dashboard Repair & Foundation

**Goal:** Fix all broken dashboard pages from v2.0 Phase 5 (LLM layer removal),
establish the foundation for IDE and agent features.

- [ ] Fix server.py — remove all 23 dead imports from deleted modules
- [ ] Fix or remove broken endpoints (task queue, model routing, provider health)
- [ ] Ensure all existing pages render: Status, Tools, Skills, Memory, Settings
- [ ] Remove stale pages: Agents (will be rebuilt), Teams (replaced by agent system)
- [ ] Add "IDE" nav item (placeholder)
- [ ] Add "Agents" nav item (placeholder for new agent management)
- [ ] Update status page to show MCP server health + node status

## Phase 2: Web IDE — Code Editor

**Goal:** Replace the "Code" page with a Monaco-based code editor.

- [ ] Integrate Monaco Editor (same engine as VS Code)
- [ ] File tree sidebar using `agent42_list_dir` + `agent42_read_file`
- [ ] File open/save via `agent42_read_file` / `agent42_write_file`
- [ ] Syntax highlighting for Python, JS, JSON, YAML, Markdown
- [ ] Git status indicators in file tree (modified, new, deleted)
- [ ] Multi-tab editing
- [ ] Search across files via `agent42_grep`

## Phase 3: Web IDE — Terminal

**Goal:** Add a web terminal for shell access.

- [ ] Integrate xterm.js for web terminal
- [ ] WebSocket connection to backend shell process
- [ ] Connect to local node or remote node (selectable)
- [ ] Command history and autocomplete
- [ ] Multiple terminal tabs (like VS Code)
- [ ] Terminal output preserved across page navigation

## Phase 4: Web IDE — Claude Code Chat Integration

**Goal:** Embed Claude Code chat in the web IDE, connected to Agent42's MCP tools.

- [ ] Chat panel (right sidebar, resizable)
- [ ] Connect to Anthropic API using user's API key or subscription
- [ ] MCP tool access — chat can call all Agent42 tools
- [ ] Memory integration — associative recall on each message
- [ ] Context awareness — knows which file is open, current project
- [ ] Conversation history persisted in Agent42 memory
- [ ] Provider switching — user selects Claude (subscription), Synthetic, OpenRouter
- [ ] Headless mode — same chat backend runs without UI for automation

## Phase 5: Agent Creation System

**Goal:** Users can create custom agents with specific tools, skills, and behaviors.

- [ ] Agent definition format (YAML/JSON):
  ```yaml
  name: support-agent
  description: Handles customer support tickets
  tools: [web_fetch, http_request, memory, template, email]
  skills: [support, communication, troubleshooting]
  memory_scope: project/support
  schedule: always  # or cron expression
  provider: synthetic  # which AI provider to use
  max_iterations: 10
  approval_required: false  # for autonomous operation
  ```
- [ ] Agent creation wizard in dashboard UI
- [ ] Agent templates: Support, Marketing, DevOps, Content, Research, Monitor
- [ ] Each agent gets isolated memory scope
- [ ] Agent-specific tool permissions (sandbox within sandbox)
- [ ] Agent behavior profiles (tone, approach, escalation rules)

## Phase 6: Agent Runtime & Orchestration

**Goal:** Agents run autonomously on local or VPS nodes.

- [ ] Agent runtime engine — launches agents as processes
- [ ] Task queue per agent — incoming work items
- [ ] Agent scheduling — cron-based, event-triggered, or always-on
- [ ] Provider selection per agent (CC subscription, Synthetic, OpenRouter)
- [ ] Agent communication — agents can pass tasks to other agents
- [ ] Iteration engine — agent works until goal met or max iterations
- [ ] Human-in-the-loop checkpoints — agent pauses for approval at defined points
- [ ] Resource limits — token budget, time budget, API call limits per agent

## Phase 7: Agent Monitoring & Management Dashboard

**Goal:** Real-time visibility into agent activity.

- [ ] Agent status dashboard — running, paused, completed, failed
- [ ] Activity feed — what each agent is doing in real-time
- [ ] Token usage per agent — cost tracking
- [ ] Memory browser per agent — what each agent knows
- [ ] Log viewer — full conversation history per agent
- [ ] Start/stop/pause controls
- [ ] Agent performance metrics — tasks completed, success rate, avg time
- [ ] Alerts — agent failures, budget exceeded, approval needed

## Phase 8: Multi-Project Agent Deployment

**Goal:** Agents organized by project, deployable to any node.

- [ ] Project workspace — group agents, files, memory by project
- [ ] Deploy agents to local node or VPS node
- [ ] Agent migration — move running agent between nodes
- [ ] Project templates — "E-commerce" includes support + marketing + dev agents
- [ ] Shared memory between project agents (with isolation from other projects)
- [ ] Project dashboard — overview of all agents for a project

## Phase 9: CLI-Anything Integration

**Goal:** Agents can control ANY software that has a codebase — GIMP, Blender,
LibreOffice, Audacity, OBS, and hundreds more via auto-generated CLI wrappers.

- [ ] Install CLI-Anything framework (`pip install cli-anything`)
- [ ] Create `CLIBridgeTool` — discovers installed CLI wrappers on the node
- [ ] Each discovered CLI registers as a sub-tool with command schema
- [ ] Agent configs can include CLI tools (e.g., `cli_libreoffice`, `cli_gimp`)
- [ ] Auto-import SKILL.md files generated by CLI-Anything into Agent42 skills
- [ ] Test with key applications: LibreOffice, GIMP, Blender, Audacity
- [ ] Document "Add Any Software" workflow for non-technical users

**Why:** Turns Agent42 from "controls code and servers" to "controls everything."
A marketing agent can create real PDFs. A content agent can edit real images.
A DevOps agent can manage OBS streaming. All autonomously.

Reference: https://github.com/HKUDS/CLI-Anything

## Technical Stack

| Component | Technology |
|-----------|-----------|
| Code Editor | Monaco Editor (VS Code engine) |
| Terminal | xterm.js + WebSocket |
| Chat/AI | Anthropic API + MCP tools |
| Agent Runtime | Python asyncio + subprocess |
| Dashboard | FastAPI + vanilla JS (existing) |
| Memory | Qdrant + ONNX embeddings (existing) |
| Scheduling | APScheduler or built-in cron |
| Auth | JWT + bcrypt (existing) |

## Provider Strategy

| Use Case | Provider | Why |
|----------|----------|-----|
| Interactive IDE chat | Claude Code Subscription | Best reasoning, human in loop |
| Autonomous agents | Synthetic.new ($30/mo) | Designed for API/agent use |
| Multi-model experiments | OpenRouter | Access to 200+ models |
| Enterprise | Bedrock / Vertex | Compliance, data residency |

## Success Criteria

- User can create a custom agent in < 5 minutes via dashboard
- Agent runs autonomously on VPS for 24+ hours without intervention
- Web IDE provides 80% of VS Code functionality
- Multiple agents can run concurrently without resource conflicts
- Agent memory is isolated but queryable across projects
- Dashboard shows real-time agent activity with < 2s latency
