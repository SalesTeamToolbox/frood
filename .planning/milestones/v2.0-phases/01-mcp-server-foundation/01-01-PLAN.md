# Phase 1: MCP Server Foundation — Plan

## Tasks

### Task 1: Add `mcp` SDK dependency
- Add `mcp` to `requirements.txt`
- Verify installation: `pip install mcp`

### Task 2: Add `to_mcp_schema()` to Tool ABC
- File: `tools/base.py`
- Add method to `Tool` class that converts to MCP `types.Tool` format
- Maps: `name` → `agent42_{name}`, `description` → description, `parameters` → `inputSchema`
- `ExtendedTool` inherits this automatically (calls merged properties)

### Task 3: Create `mcp_server.py` (main entry point)
- New file at project root: `mcp_server.py`
- Uses low-level `mcp.server.lowlevel.Server` API
- `@server.list_tools()` — iterates ToolRegistry, returns `list[types.Tool]`
- `@server.call_tool()` — receives `(name, arguments)`, strips `agent42_` prefix, calls `registry.execute()`
- Initializes: Settings, WorkspaceSandbox, CommandFilter, RateLimiter
- Registers PoC tools: ReadFileTool, WriteFileTool, ShellTool (with security layers)
- Runs with `mcp.server.stdio.stdio_server()` transport
- Standalone: `python mcp_server.py`

### Task 4: Create `mcp_registry.py` (adapter layer)
- New file: `mcp_registry.py`
- `MCPToolAdapter` class — bridges ToolRegistry ↔ MCP server
- Handles: tool name prefixing (`agent42_`), ToolResult → MCP TextContent conversion
- Wraps security layers (sandbox, command_filter) around execution
- Error responses include explanation text (per CONTEXT.md decision)

### Task 5: Register PoC tools with security
- Instantiate ReadFileTool, WriteFileTool, ShellTool with:
  - `WorkspaceSandbox(workspace_path, enabled=True)`
  - `CommandFilter()` for shell
  - `ToolRateLimiter()` wrapping execution
- Workspace path from `AGENT42_WORKSPACE` env var or cwd

### Task 6: Migrate `tools/mcp_client.py` to SDK
- Replace raw JSON-RPC with `mcp.client.stdio.stdio_client` + `ClientSession`
- Replace `MCPConnection._connect_stdio()` with `StdioServerParameters`
- Replace `MCPConnection.list_tools()` with `session.list_tools()`
- Replace `MCPConnection.call_tool()` with `session.call_tool()`
- Keep `MCPToolProxy` and `MCPManager` interfaces intact (internal consumers unchanged)
- Keep command validation allowlist security

### Task 7: Generate `.mcp.json`
- Create `.mcp.json.example` in project root (not actual .mcp.json — user configures)
- Template with `${workspaceFolder}` for AGENT42_WORKSPACE
- Document in README section

### Task 8: Tests
- `tests/test_mcp_server.py` — server tool listing, tool execution, security blocking
- `tests/test_mcp_registry.py` — adapter, name prefixing, result conversion
- `tests/test_mcp_client_sdk.py` — migrated client still works
- Update `tests/test_mcp.py` if it exists

### Task 9: Integration test
- Start MCP server as subprocess
- Connect with SDK client
- List tools → verify agent42_ prefix
- Call agent42_read_file → verify sandbox works
- Call agent42_shell with safe command → verify output
- Call agent42_shell with blocked command → verify error with explanation
