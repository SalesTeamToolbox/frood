     - Call `stripe.checkout.Session.create_async()` with:
       - `mode="payment"`
       - `line_items=[{"price_data": {"currency": "usd", "unit_amount": amount_cents, "product_data": {"name": product_name}}, "quantity": 1}]`
       - `payment_intent_data={"metadata": {"user_id": str(user_id), "type": "product_purchase", "variant_id": str(variant_id), "design_id": design_id or ""}}`
       - `metadata={"user_id": str(user_id), "type": "product_purchase"}`

   - `async def handle_checkout_completed(db: aiosqlite.Connection, event_id: str, event_type: str, session: dict) -> dict`:
     - **Idempotency check (PAY-06):** Query `stripe_events_processed` for `event_id`. If found, return `{"status": "already_processed"}`.
     - Read `session["metadata"]["type"]` to determine handler.
     - If `type == "wallet_topup"`:
       - `user_id = int(session["metadata"]["user_id"])`
       - `amount_usd = float(session["metadata"].get("amount_usd", "0"))` — or derive from `session["amount_total"] / 100`
       - Credits to add: `int(amount_usd)` (1 credit per dollar — simple v1 mapping)
       - Call `await credit_credits(db, user_id, credits, "topup", stripe_pi_id=session.get("payment_intent"))`
     - If `type == "product_purchase"`:
       - `user_id = int(session["metadata"]["user_id"])`
       - Call `await credit_credits(db, user_id, settings.purchase_credit_award, "purchase_bonus", stripe_pi_id=session.get("payment_intent"))`
     - **Record processed event** in same logical scope:
       - `await db.execute("BEGIN IMMEDIATE")`
       - `await db.execute("INSERT INTO stripe_events_processed (stripe_event_id, event_type) VALUES (?, ?)", (event_id, event_type))`
       - `await db.execute("COMMIT")`
     - Return `{"status": "ok"}`

   **IMPORTANT:** The idempotency INSERT and credit mutation should ideally be in the same transaction. Since credit_credits already uses its own BEGIN IMMEDIATE, the approach is: (1) check idempotency, (2) call credit_credits (its own tx), (3) record processed event (separate tx). If crash between 2 and 3, Stripe retries and credit_credits runs again — but the append-only ledger makes this visible and auditable. For Phase 1 this is acceptable. A single-transaction approach would require refactoring wallet_service to accept an existing transaction, which can be done in a future optimization.

   - Set `stripe.api_key = settings.stripe_secret_key` at module load via a lazy init pattern, or set it in the lifespan function in main.py (preferred — see Task 2).

   Provides endpoints for Stripe Checkout Session creation (wallet top-up and product purchase)
   and the Stripe webhook handler. The webhook endpoint uses HMAC signature verification
   instead of JWT authentication (PAY-08).

   - Import: `stripe`, `Request`, `APIRouter`, `Depends`, `HTTPException`, `status`, `aiosqlite`, `get_db`, `get_current_user`, `order_service`, `settings`, `logging`
   - Define `router = APIRouter()`

   - Pydantic request/response models:
     ```python
     class TopupRequest(BaseModel):
         amount_usd: float  # Minimum $5.00 (PAY-01)
         success_url: str = "http://localhost:8001/?payment=success"
         cancel_url: str = "http://localhost:8001/?payment=cancelled"

     class PurchaseRequest(BaseModel):
         variant_id: int
         design_id: str | None = None
         success_url: str = "http://localhost:8001/?payment=success"
         cancel_url: str = "http://localhost:8001/?payment=cancelled"

     class CheckoutResponse(BaseModel):
         checkout_url: str
     ```

   - `POST /topup` endpoint (PAY-01 — protected, requires auth):
     ```python
     @router.post("/topup", response_model=CheckoutResponse)
     async def create_topup(
         body: TopupRequest,
         current_user: dict = Depends(get_current_user),
         db: aiosqlite.Connection = Depends(get_db),
     ) -> CheckoutResponse:
         try:
             url = await order_service.create_topup_session(
                 user_id=current_user["id"],
                 amount_usd=body.amount_usd,
                 success_url=body.success_url,
                 cancel_url=body.cancel_url,
             )
             return CheckoutResponse(checkout_url=url)
         except ValueError as e:
             raise HTTPException(status_code=status.HTTP_400_BAD_REQUEST, detail=str(e))
     ```

   - `POST /purchase` endpoint (PAY-05 — protected, requires auth):
     ```python
     @router.post("/purchase", response_model=CheckoutResponse)
     async def create_purchase(
         body: PurchaseRequest,
         current_user: dict = Depends(get_current_user),
         db: aiosqlite.Connection = Depends(get_db),
     ) -> CheckoutResponse:
         # Look up variant to get price and product name
         cursor = await db.execute(
             """SELECT pv.retail_price, p.name FROM product_variants pv
                JOIN products p ON p.id = pv.product_id
                WHERE pv.id = ? AND pv.in_stock = 1""",
             (body.variant_id,)
         )
         variant = await cursor.fetchone()
         if not variant:
             raise HTTPException(status_code=404, detail="Variant not found or out of stock")

         url = await order_service.create_purchase_session(
             user_id=current_user["id"],
             variant_id=body.variant_id,
             retail_price_usd=variant[0],
             product_name=variant[1],
             design_id=body.design_id,
             success_url=body.success_url,
             cancel_url=body.cancel_url,
         )
         return CheckoutResponse(checkout_url=url)
     ```

   - `POST /webhook/stripe` endpoint (PAY-06, PAY-08 — NO auth dependency, HMAC verification):
     ```python
     @router.post("/webhook/stripe")
     async def stripe_webhook(
         request: Request,
         db: aiosqlite.Connection = Depends(get_db),
     ) -> dict:
         """
         Stripe webhook handler. Authenticates via HMAC signature (PAY-08).
         Idempotent — duplicate events are no-ops (PAY-06).
         NO get_current_user dependency — this endpoint is called by Stripe, not a user.
         """
         payload = await request.body()
         sig_header = request.headers.get("stripe-signature", "")

         try:
             event = stripe.Webhook.construct_event(
                 payload, sig_header, settings.stripe_webhook_secret
             )
         except stripe.error.SignatureVerificationError:
             raise HTTPException(status_code=400, detail="Invalid webhook signature")
         except ValueError:
             raise HTTPException(status_code=400, detail="Invalid payload")

         event_id = event["id"]
         event_type = event["type"]

         if event_type == "checkout.session.completed":
             session = event["data"]["object"]
             result = await order_service.handle_checkout_completed(
                 db=db, event_id=event_id, event_type=event_type, session=session
             )
             return result

         return {"status": "unhandled_event_type"}
     ```

   **CRITICAL (PAY-08):** The `stripe_webhook` function does NOT have `current_user: dict = Depends(get_current_user)` — it uses `request.body()` + `stripe.Webhook.construct_event()` for authentication. This is intentional because Stripe calls this endpoint, not a browser with a JWT.

   **CRITICAL (Pitfall 1 from research):** The webhook handler uses `await request.body()` to get raw bytes for HMAC verification. It does NOT use a Pydantic model parameter for the body — that would cause FastAPI to parse the JSON and the raw bytes would be lost.
