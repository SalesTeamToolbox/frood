2. Create `apps/meatheadgear/services/wallet_service.py`:
   - Define `class InsufficientCreditsError(Exception): pass`
   - Implement `async def get_balance(db: aiosqlite.Connection, user_id: int) -> int`:
     ```python
     cursor = await db.execute(
         "SELECT balance_after FROM wallet_ledger WHERE user_id = ? ORDER BY id DESC LIMIT 1",
         (user_id,)
     )
     row = await cursor.fetchone()
     return row[0] if row else 0
     ```
   - Implement `async def credit_credits(db, user_id, amount, reason, stripe_pi_id=None) -> int`:
     Uses `BEGIN IMMEDIATE` transaction. Reads current balance (or 0), computes new_balance = current + amount, INSERTs row with delta_credits=+amount and balance_after=new_balance, COMMITs, returns new_balance. On exception: ROLLBACK and re-raise.
   - Implement `async def debit_credits(db, user_id, amount, reason, design_id=None) -> int`:
     Uses `BEGIN IMMEDIATE` transaction. Reads current balance. If balance < amount: ROLLBACK, raise InsufficientCreditsError with message including current balance and requested amount. Otherwise: new_balance = current - amount, INSERT row with delta_credits=-amount and balance_after=new_balance, COMMIT, return new_balance.
   - All functions take `db: aiosqlite.Connection` as first parameter (passed from router, NOT calling get_db() themselves — per Pitfall 5 in research).
